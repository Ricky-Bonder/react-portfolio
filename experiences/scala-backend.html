High-Throughput Data Acquisition Service

Reactive stream processing using Scala and Akka Actors.

The Overview

A specialized backend service designed to ingest, process, and forward high-frequency data streams. Utilizing the Actor Model via Scala and Akka, the system was architected to handle concurrency natively, ensuring that failures in one data stream (e.g., a specific sensor) would not crash the entire system. The service acted as the reliability layer, buffering data during network instability and guaranteeing delivery via MQTT.

Engineering Challenges

Concurrency & Fault Tolerance

Managing simultaneous data streams required a non-blocking architecture. The Akka Actor system allowed for 'Let it Crash' fault tolerance, isolating failures to individual actors.

Algorithm Optimization

Implemented custom buffering algorithms to handle backpressure when the upstream network was slower than the ingestion rate.

Key Responsibilities

  • Developed robust data acquisition services using the Actor Model.
  • Ensured reliable communication via Serial and MQTT protocols.
  • Designed APIs for internal communication between system components.
